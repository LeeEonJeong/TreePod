    JavaScript에 대해 공부하자! -4 JavaScript / Study

    2014.05.24. 23:11

    http://blog.naver.com/nu_essy/220009649288

    번역하기 전용뷰어 보기

     

     이번 글에서는 자바스크립트의 기본구조에 대해서 익혀 보도록 하겠습니다 !

     먼저, 다음과 같은 예제 문을 확인해 봅시다.

     

     

      자바 스크립트는 위와 같이 <script> 태그를 사용하여 HTML 문서에 코드를 작성합니다.

     <script>는 html 태그 밑에 위치하구요!

      여기서 태그(Tag)라는 것은 어떤 문자에 의미가 부여되도록 만드는 것이라고 생각하시면 됩니다!.

      1번줄의 <!DOCTYPE html>은 이 해당 문서를 html5 대한 문서로 만들어 버립니다 !

     

      html에서는 위에서부터 아래로 절차적으로 해석해서 나가기때문에, html -> head -> body -> body내부의 script -> 종료 

     이런식으로 순차적으로 해석해 나갑니다. 이 순차적인 해석으로 인해 문제가 발생할 수 있습니다.

     다음 코드를 쳐보도록 하죠!

     아!, 다음과 같은 코드는 크롬 console에 직접칠 필요없이 다음과 같은 방법으로 하시면됩니다!

     먼저, notepad를 연 이후에 아래의 코드를 작성하신 이후 저장할때, 파일명.html로 저장하고 실행시켜줍니다!

     

     

     

     

     

    아래의 코드를 입력해보도록 하죠!

     

     

     

    <!DOCTYPE html>
    <html>
    <head>
    <title> 문서의 제목을 입력합니다.</title>
    <script>
              var myDiv = document.querySelector("div"); 
              myDiv.innerHTML = "반갑습니다";
    </script>
    </head>
     
    <body>
                <!-- 브라우저 화면에 보여지는 내용입니다. -->
                <div>안녕하세요</div>
    </body>
    </html>

     

     

     

     

     위의 코드를 보시면, head 태그 안에 script가 속해 있는 것을 확인 할 수 있습니다. 위에서 말했듯이 head 태그 이후에 body 태그로 

    실행되게 되는데.. 과연 이대로 작성하면 어떠한 일이 일어날까요?

     

     

     

     

     

     위에서는 '반갑습니다'가 나와야 할 것 같은데, '안녕하세요'만 나오고, F12 버튼을 누르게 되면

     Uncaught TypeError: Cannot set property 'innerHTML' of null 이라는 에러가 발생하게 됩니다.

     해당 에러의 의미는 소스코드 상에서 innerHTML이라는 속성을 null에 추가할 수 없다라는 의미로, 

     6줄에서 선언한 myDiv가 아무것도 가르키지 않고 있는 상태라는 것을 알 수 있습니다. 

     
     6번째 줄을 보면 selector를 실행하였을때 div라는 태그를 이용하는데, 이 tag를 찾지 못해서 null값을 저장시켰다는 것을 알 수 있습니다!

     이는 body에 선언된 div까지 가기도 전에 자바 스크립트가 먼저 실행되어버려서 찾지 못하는 경우입니다.

     

     

     dom틀에 div태그가 만들어지기전 (html문서를 완성시키기전에) 자바스크립트가 먼저 생성된 것으로

     간단히 말하면 html 읽기를 완료하기전에 자바스크립트가 먼저 실행되어 버린것이죠 

     그렇다면 어떻게 해야할까요? 

     위의 진행 순서에서 body 분에 div가 선언되어있으므로 다음과 같이 script의 위치를 바꾸는 방법이 존재합니다! 

     

     

    <!DOCTYPE html> 
    <html>
    <head>
    <title> 문서의 제목을 입력합니다.</title>
    </head>
    <body>
          <!-- 브라우저 화면에 보여지는 내용입니다. -->
          <div>안녕하세요</div>
          <script>
              var myDiv = document.querySelector("div");  

              myDiv.innerHTML = "반갑습니다";
          </script>
    </body>
    </html>

     

     

    이번 것을 쳐보면 정상적으로 '반갑습니다'가 뜨는 것을 확인 할 수 있습니다.

    그러나 이렇게 body안에 script를 쑤셔넣는 경우 가독성이 떨어지는 좋은 코딩이라 

    할 수 없으므로 좀더 좋은 방법을 생각해야 합니다. 이번엔 function을 쓰는 방법을 구현해 보죠!

     

     


    <!DOCTYPE html> 
    <html>
    <head>
          <title> 문서의 제목을 입력합니다.</title>
          <script>
                function init(){
                    var myDiv = document.querySelector("div");  
                    myDiv.innerHTML = "반갑습니다";
                }      
          </script>
    </head>
    <body>
    <!-- 브라우저 화면에 보여지는 내용입니다. -->
    <div>안녕하세요</div>
    </body>

    </html> 

     

    자, 이렇게 하면 어떤가요?!  네! 당연히 안되죠 ㅋㅋㅋㅋㅋ 

    이렇게 하면 처음과 같은 에러화면을 발견할 수 있습니다. 

     보시면 init이란 함수를 선언 했는데, 실제로 쓰인부분이 없기 때문이죠!

     그럼 어떻게 써야 하는걸까요?!?!

     onload를 써주게 됩니다

     

     

    <!DOCTYPE html> 
    <html>
    <head>
    <title> 문서의 제목을 입력합니다.</title>
    <script>
    function init(){
        var myDiv = document.querySelector("div");  
            myDiv.innerHTML = "반갑습니다";
    }
    </script>
    </head>
    <body onload ="init();">
    <!-- 브라우저 화면에 보여지는 내용입니다. -->
    <div>안녕하세요</div>
    </body>
    </html>

     

     

     정상으로 동작하는 것을 확인 할 수 있습니다! . 여기서 onload라는 것은 자바스크립트가 인터프리터 방식으로 위에서부터

     먼저 실행하기 때문에 script가 먼저 실행될 경우 body의 객체를 사용할 수 없으므로 이러한 '이벤트'를 사용하여 body의 객체를

     자바 스크립트가 사용할 수 있도록 만들어 주는 것이죠! 

     

     onload는 "페이자가 다 로드되면 init이라는 method를 호출하라!" 이렇게 요약할수 있죠 !

     

     하지만 html문서는 이런식으로 되는 형식을 좋아하지 않습니다..(까다롭죠 ) 

     따라서 별도의 자바스크립트 파일을 만들어 불러오는 방식으로 구성해 보겠습니다!

     

     

     별도의 자바스크립트 파일을 만들기위해 메모장 새로 열기하여 function 내부의 내용을 저장하고 

     

     default.js라는 이름으로 저장합니다! 확장자는 자바스크립트의 줄임말인걸 알수 있죠?!

    simple.html의 경우 function부분과 onload부분을 지우고,  스크립트에 default.js를 참조한다는 내용을 작성합니다

     

    <default.js 파일의 내용 >

    var myDiv = document.querySelector("div");
    myDiv.innerHTML = "반갑습니다";

     

     

    <simple.html 파일의 내용 >

    <!DOCTYPE html>
    <html>
    <head>
    <title> 문서의 제목을 입력합니다.</title>
    <script src="default.js">
    </script>
    </head>
     
    <body>
    <!-- 브라우저 화면에 보여지는 내용입니다. -->
    <div>안녕하세요</div>
    </body>
    </html>

     

     

    이렇게 작성하시면 아까와 같은 아래의 에러가 발생하게 됩니다.


    Uncaught TypeError: Cannot set property 'innerHTML' of null

     

     이 역시 body의 div를 제대로 참조하지 못해서 발생한 것으로, 다음과 같이 onload와 function을 적절히 사용히주시면

     

     우리가 원하는 최종적인 결과를 얻을 수 있습니다!

     

    <default.js 파일의 내용 >

    function init(){
    var myDiv = document.querySelector("div");  
    myDiv.innerHTML = "반갑습니다";
    }

     

    <simple.html 파일의 내용 >

    <!DOCTYPE html> 
    <html>
    <head>
    <title> 문서의 제목을 입력합니다.</title>
    <script src="default.js">
    </script>
    </head>

    <body onload ="init();">
    <!-- 브라우저 화면에 보여지는 내용입니다. -->
    <div>안녕하세요</div>
    </body>
    </html>

     

     

    자 여기까지만 해도 원하는 결과를 얻을 수 있었습니다!

    그럼 마지막으로 'defer'라는 것을 사용해서 원하는 결과를 얻어보도록 하죠!

     

     

     처음에 언급한것처럼 head부분에 자바 스크립트가 존재하고, body쪽 객체를 참조하려는 경우 에러가 발생하는 것을 확인 할 수 

    있었는데요, 이는 html문서가 실행을 완료하기 전에 자바 스크립트가 먼저 작업을 완료하고 실행시켜 버려서 발생하는 문제였습니다.

     

     

     위의 실행도는  'defer'를 설명하기위하여 작성한 것으로 이를바탕으로 쉽게 설명드리도록 하겠습니다.

     

    우선 HTML을 파싱하는 과정을 진행하구요 이때 스크립트 태그로 이동하게 됩니다. 이때부터 HTML문서는 실행중인 상태가 되구요, 스크립트 태그를 읽은 후에 네트워크 프로세서를 통하여 자바스크립트가 실행되어집니다.

     


    1의 경우는 스크립트 태그가 실행할 준비를 마치고 자바스크립트가 실행되는 단계입니다. 원래 기존의 경우 바로 실행되어 에러를 발생 시키지만 'defer'를 사용하는 경우에는 바로 자바 스크립트를 실행시키지 않고 대기하게 됩니다.


    2는 이제 HTML 문서가 실행을 완료한 시점을 나타낸 것으로 이때 JavaScript는 대기 상태를 멈추고 JavaScript를 

    실행하게 됩니다. 이과정을 통하여 에러 없이 수행하도록 합니다!

     

    이외에도 async 속성이 있는데 이는 스크립트 다운로드가 완료되면 바로 자바 스크립트를 실행하지만 worker라는 것을 통하여 비동기 

    적으로 실행하여 에러를 방지한다고 합니다.

     

     'defer'를 사용한 코드는 다음과 같습니다.

     

     

     

    <default.js 파일의 내용 >

    function init(){
    var myDiv = document.querySelector("div");  
    myDiv.innerHTML = "반갑습니다";
    }
    init();

     

    <simple.html 파일의 내용 >

    <!DOCTYPE html> 
    <html>
    <head>
    <title> 문서의 제목을 입력합니다.</title>
    <script src="default.js" defer>
    </script>
    </head>

    <body>
    <!-- 브라우저 화면에 보여지는 내용입니다. -->
    <div>안녕하세요</div>
    </body>
    </html>

     

     

     default.js파일에는 마지막에 함수를 호출하는 부분이 추가되었고, simple.html에서는 스크립트에서

     src = 'default.js"뒤에 'defer'가 붙은 것을 확인 할 수 있습니다! 에러없이 원하는 결과를 수행 할 수 있었습니다!

     

     

     

     실습위주의 글이었다보니 글이 많이 길어졌네요!

     

     오늘은 여기까지 올리고 내일 이어서 올리도록 하겠습니다. 감사합니다!
